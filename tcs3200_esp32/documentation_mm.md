# ESP32 တွင် TCS3200 Color Sensor + SSD1306 OLED အသုံးပြုခြင်း

TCS3200 color sensor ကို အသုံးပြု၍ အရောင်များကို ဖတ်ရှုပြီး၊ ဖတ်ရှုရရှိသော အရောင်အမည်နှင့် RGB တန်ဖိုးများကို OLED မျက်နှာပြင်ငယ်ပေါ်တွင် ပြသပေးမည့် စတင်လေ့လာသူများအတွက် အထူးသင့်လျော်သော Arduino ပရောဂျက်တစ်ခု ဖြစ်ပါသည်။

## ဤပရောဂျက်က ဘာလုပ်ပေးသလဲ 

- TCS3200 ဆင်ဆာကို အရောင်ရှိသော အရာဝတ္ထုဆီသို့ ချိန်ရွယ်ပါသည်။
- မျက်နှာပြင်မှ ရောင်ပြန်ဟပ်လာသော အနီ (Red)၊ အစိမ်း (Green) နှင့် အပြာ (Blue) အလင်းတန်ဖိုးများကို ဖတ်ရှုပါသည်။
- အဓိက အရောင်ကို တွက်ချက်ပေးပါသည်။
- အရောင်အမည် (ဥပမာ - RED, GREEN, BLUE, YELLOW, WHITE) ကို OLED မျက်နှာပြင်ပေါ်တွင် စာလုံးအကြီးဖြင့် ပြသပေးပါသည်။
- R, G, B ဂဏန်းတန်ဖိုးများ (၀ မှ ၂၅၅ အထိ) ကို အရောင်အမည်၏ အောက်တွင် ပြသပေးပါသည်။
- ၎င်းအချက်အလက်များကို Arduino Serial Monitor တွင်လည်း ပြသပေးပါသည်။

## ဝါယာကြိုး ချိတ်ဆက်မှုများ

### TCS3200 Color Sensor → ESP32

| TCS3200 Pin | ESP32 Pin | ဖော်ပြချက် |
|-------------|-----------|-------------|
| VCC | 3V3 | ပါဝါ (3.3 V) |
| GND | GND | Ground |
| S0 | D5 (GPIO 5) | Frequency scaling bit 0 |
| S1 | D18 (GPIO 18) | Frequency scaling bit 1 |
| S2 | D19 (GPIO 19) | Color filter select bit 0 |
| S3 | D15 (GPIO 15) | Color filter select bit 1 |
| OUT | D4 (GPIO 4) | Frequency output (ဆင်ဆာ ဒေတာ) |
| LED (OE) | D32 (GPIO 32) | အလင်းပေး LED အဖွင့်/အပိတ် ထိန်းချုပ်မှု |

> **S0/S1 ဆိုတာ ဘာလဲ?** ၎င်းတို့သည် ဆင်ဆာမှ pulse များ မည်မျှမြန်မြန် ထုတ်လွှတ်မည်ကို သတ်မှတ်ပေးပါသည်။ ဤကုဒ်တွင် 20% scaling (S0=HIGH, S1=LOW) ကို အသုံးပြုထားပြီး ၎င်းသည် အမြန်နှုန်းနှင့် တိကျမှုကြား အကောင်းဆုံး ချိန်ခွင်လျှာဖြစ်ပါသည်။
>
> **S2/S3 ဆိုတာ ဘာလဲ?** အချိန်တစ်ခုတွင် မည်သည့် အရောင်ဖစ်တာ (အနီ၊ အစိမ်း၊ အပြာ၊ သို့မဟုတ် အကြည်) ကို ဖတ်ရှုမည်ကို ရွေးချယ်ပေးပါသည်။ ကုဒ်က ၎င်းတို့ကို အလိုအလျောက် ပြောင်းလဲပေးပါသည်။
>
> **OUT pin:** ဆင်ဆာသည် အလင်းပြင်းအားကို square wave အဖြစ်သို့ ပြောင်းလဲပေးပါသည်။ ESP32 သည် pulse တစ်ခုစီ၏ ကြာချိန်ကို တိုင်းတာပြီး၊ pulse ပိုတိုလေ ထိုအရောင်၏ အလင်းပိုများလေ ဖြစ်သည်။

---

### SSD1306 OLED Display → ESP32

| OLED Pin | ESP32 Pin | ဖော်ပြချက် |
|----------|-----------|-------------|
| VCC | 3V3 | ပါဝါ (3.3 V) |
| GND | GND | Ground |
| SDA | D21 (GPIO 21) | I2C ဒေတာ လိုင်း |
| SCL | D22 (GPIO 22) | I2C ကလော့ခ် (clock) လိုင်း |

> **I2C ဆိုတာ ဘာလဲ?** ၎င်းသည် ဒေတာပေးပို့ရန် ဝါယာကြိုး ၂ ချောင်း (SDA နှင့် SCL) ကိုသာ အသုံးပြုသော ဆက်သွယ်ရေး ပရိုတိုကော ဖြစ်သည်။ GPIO 21 နှင့် GPIO 22 တို့သည် ESP32 ဘုတ်အများစုရှိ ပုံသေ I2C pin များ ဖြစ်ကြသည်။
>
> **I2C Address:** ဤကုဒ်သည် SSD1306 OLED များအတွက် အသုံးအများဆုံး လိပ်စာဖြစ်သော `0x3C` ကို အသုံးပြုထားသည်။ အကယ်၍ သင့် display မပွင့်ပါက `0x3D` ကို စမ်းကြည့်ပါ (ပြဿနာ ဖြေရှင်းခြင်း ကဏ္ဍကို ကြည့်ပါ)။

| ESP32 Pin | ချိတ်ဆက်ရမည့် နေရာ |

## မည်သို့ အလုပ်လုပ်သနည်း

### အဆင့် ၁ — Setup ပြုလုပ်ခြင်း

- ESP32 သည် TCS3200 ၏ ထိန်းချုပ် pin များ (S0–S3) ကို output များအဖြစ် လည်းကောင်း၊ OUT pin ကို input အဖြစ် လည်းကောင်း သတ်မှတ်ပေးသည်။
- S0 ကို HIGH နှင့် S1 ကို LOW ထားခြင်းဖြင့် Frequency scaling ကို 20% တွင် သတ်မှတ်သည်။
- OLED ကို I2C မှတစ်ဆင့် အစပြု (initialize) ပြီး "TCS3200 / Color Sensor" ဟူသော splash screen ကို ၂ စက္ကန့်ကြာ ပြသပေးသည်။

### အဆင့် ၂ — အရောင် Channel ကို ဖတ်ရှုခြင်း

`readChannel(s2, s3)` ဖန်ရှင်သည် S2 နှင့် S3 ကို ပြောင်းလဲသတ်မှတ်ခြင်းဖြင့် အရောင် filter တစ်ခုကို ရွေးချယ်ကာ၊ ဆင်ဆာ ငြိမ်သွားစေရန် 50 ms စောင့်ဆိုင်းပြီးနောက် `pulseIn()` ကို အသုံးပြု၍ OUT pin မှ pulse အကျယ် (ကြာချိန်) ကို တိုင်းတာသည်။

| အရောင် Filter | S2 | S3 |
|-------------|----|----|
| အနီ (Red) | LOW | LOW |
| အစိမ်း (Green) | HIGH | HIGH |
| အပြာ (Blue) | LOW | HIGH |

> **Pulse ပိုတိုလေ = ထိုအရောင်၏ အလင်းပြန်မှု ပိုများလေ ဖြစ်သည်။** အနီရောင် အရာဝတ္ထုတစ်ခုသည် အနီရောင်အလင်းကို အများအပြား ရောင်ပြန်ဟပ်စေသောကြောင့်၊ အနီရောင် channel သည် အတိုဆုံး pulse ကို ထုတ်ပေးသည်။

### အဆင့် ၃ — RGB (၀–၂၅၅) သို့ ပြောင်းလဲခြင်း

ရရှိလာသော မူရင်း pulse တန်ဖိုးများကို ၀–၂၅၅ စကေး (scale) သို့ ပြောင်းလဲပေးသည်။ အတိုဆုံး pulse (ရောင်ပြန်ဟပ်မှု အများဆုံး) ရှိသော channel ကို ၂၅၅ အဖြစ် သတ်မှတ်ပြီး ကျန်တန်ဖိုးများကို ၎င်းအပေါ်မူတည်၍ အချိုးချ ပြောင်းလဲပါသည်။

### အဆင့် ၄ — အရောင် ခွဲခြားသိမြင်ခြင်း (Color Detection)

`detectColor()` ဖန်ရှင်သည် မူရင်း pulse တန်ဖိုး သုံးခုကို စစ်ဆေးပြီး အရောင်အမည်ကို ဆုံးဖြတ်ပေးသည်-

- **WHITE (အဖြူ)** — channel သုံးခုလုံးတွင် တူညီပြီး တိုတောင်းသော pulse များ ရှိသည် (အလင်းရောင် အများအပြား ညီညာစွာ ရောင်ပြန်ဟပ်သည်)။
- **NO OBJ (အရာဝတ္ထုမရှိ)** — channel အားလုံး တူညီသော်လည်း ရောင်ပြန်ဟပ်သော အလင်း သိပ်မရှိပါ (ဆင်ဆာအနီးတွင် ဘာမှမရှိပါ)။
- **RED (အနီ)** — အနီရောင် channel တွင် အတိုဆုံး pulse ရှိပြီး၊ အစိမ်းရောင်သည် အပြာရောင်ထက် သိပ်မနိမ့်ပါ။
- **YELLOW (အဝါ)** — အနီရောင် channel အတိုဆုံးဖြစ်သော်လည်း၊ အစိမ်းရောင်သည် အပြာရောင်ထက် သိသိသာသာ ပို၍နိမ့်သည် (အနီ + အစိမ်း = အဝါ)။
- **GREEN (အစိမ်း)** — အစိမ်းရောင် channel တွင် အတိုဆုံး pulse ရှိသည်။
- **BLUE (အပြာ)** — အပြာရောင် channel တွင် အတိုဆုံး pulse ရှိသည်။

### အဆင့် ၅ — မျက်နှာပြင်တွင် ပြသခြင်း

မီလီစက္ကန့် ၃၅၀ (350 ms) တိုင်းတွင် loop က-
1. အရောင် channel သုံးခုလုံးကို ဖတ်သည်။
2. အရောင်အမည်ကို ခွဲခြားသိမြင်သည်။
3. OLED မျက်နှာပြင်ကို ရှင်းလင်းပြီး အရောင်အမည်ကို စာလုံးအကြီး (size 2) ဖြင့် ရေးသားပြသသည်။
4. အလျားလိုက် မျဉ်းတစ်ကြောင်း ဆွဲပေးသည်။
5. R, G, B တန်ဖိုးများကို ၎င်းအောက်တွင် စာလုံးအသေးဖြင့် ရေးသားပြသသည်။
6. အဆိုပါ ဒေတာများကို Serial Monitor သို့ ပေးပို့သည်။

---

## ကုဒ် (Code) ထည့်သွင်းခြင်း

1. ESP32 ကို သင်၏ ကွန်ပျူတာနှင့် USB မှတစ်ဆင့် ချိတ်ဆက်ပါ။
2. Arduino IDE တွင် `tcs3200_esp32.ino` ကို ဖွင့်ပါ။
3. **Tools → Board** သို့ သွားပြီး သင်၏ ESP32 board (ဥပမာ - **ESP32 Dev Module**) ကို ရွေးချယ်ပါ။
4. **Tools → Port** သို့ သွားပြီး မှန်ကန်သော COM port ကို ရွေးချယ်ပါ။
5. **Upload** ခလုတ် (ညာဘက်သို့ ညွှန်ပြနေသော မြှားပုံ) ကို နှိပ်ပါ။
6. အောက်ခြေဘားတွင် "Done uploading" ဟု ပေါ်လာသည်အထိ စောင့်ပါ။

> **ESP32 ကို ပထမဆုံးအကြိမ် အသုံးပြုခြင်းလား?** သင်သည် ESP32 board package ကို ဦးစွာ ထည့်သွင်းရန် လိုအပ်သည်။ **File → Preferences** သို့ သွားပါ၊ ဤ URL `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json` ကို "Additional Boards Manager URLs" တွင် ထည့်ပါ။
> ထို့နောက် **Tools → Board → Boards Manager** သို့ သွား၍ "esp32" ဟု ရှာဖွေကာ Espressif Systems မှ package ကို install လုပ်ပါ။

## Serial Monitor တွင် ပြသမည့် အချက်အလက်များ

အချိန်နှင့်တပြေးညီ အချက်အလက်များကို ကြည့်ရှုရန် Serial Monitor (**Tools → Serial Monitor**) ကို **115200 baud** ဖြင့် ဖွင့်ပါ-

```
RED     R:255 G:42 B:38  raw(312,2840,2910)
RED     R:255 G:40 B:36  raw(308,2860,2935)
GREEN   R:45 G:255 B:60  raw(2720,330,2600)
WHITE   R:255 G:240 B:230  raw(120,125,128)
NO OBJ  R:90 G:88 B:92  raw(1800,1820,1790)
```

စာကြောင်းတစ်ကြောင်းစီတွင် အောက်ပါတို့ကို ပြသထားသည်-
- တွေ့ရှိရသော အရောင်အမည်
- ပြောင်းလဲထားသော (Normalized) RGB တန်ဖိုးများ (၀–၂၅၅)
- မိုက်ခရိုစက္ကန့် (microseconds) ဖြင့် ဖော်ပြထားသော မူရင်း pulse အကျယ်များ `(rawR, rawG, rawB)`

## ဖတ်ရှုနိုင်သော အရောင်များ

| အရောင် (Color) | ဖော်ပြချက် |
|-------|-------------|
| RED | အနီရောင် အရာဝတ္ထုများ |
| GREEN | အစိမ်းရောင် အရာဝတ္ထုများ |
| BLUE | အပြာရောင် အရာဝတ္ထုများ |
| YELLOW | အဝါရောင် အရာဝတ္ထုများ (အနီ + အစိမ်း ရောင်ပြန်ဟပ်မှု) |
| WHITE | အဖြူရောင် သို့မဟုတ် အလွန်လင်းသော အရောင်ရှိသည့် အရာဝတ္ထုများ |
| NO OBJ | အရာဝတ္ထုကို မတွေ့ရှိပါ၊ သို့မဟုတ် အရာဝတ္ထုသည် ဝေးလွန်းနေပါသည် |
| UNKNOWN | မသေချာသော ဖတ်ရှုမှု |

> **အကောင်းဆုံး ရလဒ်များရရှိရန် အကြံပြုချက်များ -**
> - အရာဝတ္ထုကို ဆင်ဆာ၏ အောက်တည့်တည့် ၁ မှ ၃ စင်တီမီတာ (1–3 cm) အကွာတွင် ကိုင်ထားပါ။
> - ပြောင်လက်နေသော မျက်နှာပြင်များသည် ဆင်ဆာကို မှားယွင်းစေနိုင်သောကြောင့် ပြန့်ပြူးပြီး အရောင်မလက်သော (matte) မျက်နှာပြင်များကို အသုံးပြုပါ။
> - ပတ်ဝန်းကျင် အလင်းရောင် မှိန်ခြင်းက တိကျမှုကို ပိုမိုကောင်းမွန်စေပါသည်။
> - အကယ်၍ ဖတ်ရှုမှုများ မတည်ငြိမ်ပါက ကုဒ်အတွင်းရှိ `#define LED_ENABLE false` ကို `#define LED_ENABLE true` သို့ ပြောင်းလဲခြင်းဖြင့် ဆင်ဆာပေါ်ရှိ LED များကို ဖွင့်ပါ။

## ပြဿနာ ဖြေရှင်းခြင်း (Troubleshooting)

| ပြဿနာ | ဖြစ်နိုင်သော အကြောင်းရင်း | ဖြေရှင်းနည်း |
|---------|-------------|----------|
| OLED ဘာမှမပေါ်ဘဲ မည်းနေခြင်း | I2C address မှားယွင်းနေခြင်း | ကုဒ်ထဲတွင် `OLED_ADDRESS` ကို `0x3C` မှ `0x3D` သို့ ပြောင်းကြည့်ပါ |
| OLED ဘာမှမပေါ်ဘဲ မည်းနေခြင်း | SDA/SCL လွဲနေခြင်း | SDA → D21 နှင့် SCL → D22 မှန်ကန်မှုရှိမရှိ စစ်ဆေးပါ |
| Serial Monitor တွင် စာသားများ ရှုပ်ထွေးနေခြင်း | Baud rate မှားယွင်းနေခြင်း | Serial Monitor ကို **115200** တွင် သတ်မှတ်ပါ |
| အမြဲတမ်း "NO OBJ" ဟု ပြနေခြင်း | အရာဝတ္ထု ဝေးလွန်းနေခြင်း | အရာဝတ္ထုကို ဆင်ဆာနှင့် ၁ မှ ၃ စင်တီမီတာ အကွာသို့ ယူဆောင်လာပါ |
| အမြဲတမ်း "UNKNOWN" ဟု ပြနေခြင်း | OUT ကြိုး လွတ်နေခြင်း | GPIO 4 ပေါ်ရှိ ဝါယာကြိုးကို သေချာစွာ ပြန်စိုက်ပါ |
| Upload လုပ်၍ မရခြင်း | ဘုတ် (Board)/Port ရွေးချယ်မှု မှားယွင်းနေခြင်း | **Tools** မီနူးအောက်တွင် Board နှင့် COM port ကို စစ်ဆေးပါ |
| Upload လုပ်၍ မရခြင်း | ESP32 သည် flash mode တွင် မရှိခြင်း | Upload ကို နှိပ်စဉ် ESP32 ပေါ်ရှိ **BOOT** ခလုတ်ကို ဖိထားပါ၊ upload စတင်သောအခါမှ လွှတ်ပါ |
| ဖတ်ရှုမှုများ ဆူညံနေခြင်း/မတည်ငြိမ်ခြင်း | ပတ်ဝန်းကျင် အလင်းရောင် လင်းလွန်းနေခြင်း | ဆင်ဆာကို အပေါ်စီးမှ အလင်းရောင်များနှင့် ကင်းလွတ်အောင် ကာကွယ်ပါ သို့မဟုတ် ဆင်ဆာပေါ်ရှိ LED များကို ဖွင့်ပါ |
